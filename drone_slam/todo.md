# SLAM Pipeline Setup: Fast-LIO2, EKF, and RTAB-Map

This guide details how to integrate **Fast-LIO2** (for high-frequency LiDAR-Inertial Odometry) and an **Extended Kalman Filter (EKF)** into our ROS 2 autonomous drone mapping pipeline.

> Due to ROS 2 custom message generation constraints, we must build the Livox LiDAR drivers and Fast-LIO2 in **separate, isolated workspaces** to avoid race conditions during compilation.

---

## Phase 1: The Livox Driver Workspace

Fast-LIO2 requires custom message structures generated by the Livox driver to process point clouds. This must be built and sourced before Fast-LIO2 is introduced.

**1. Create the workspace:**
```bash
mkdir -p ~/ws_livox/src
cd ~/ws_livox/src
```

**2. Clone the Livox ROS 2 Driver:**
```bash
git clone https://github.com/Livox-SDK/livox_ros_driver2.git
```

**3. Build the driver:**
```bash
cd ~/ws_livox
colcon build --symlink-install
```

**4. Source the workspace:**
```bash
source ~/ws_livox/install/setup.bash
```
*(This registers the custom LiDAR messages with your terminal so the next step won't fail.)*

---

## Phase 2: The Fast-LIO2 Workspace

Now that the Livox messages are registered, we can safely build the ROS 2 port of Fast-LIO2.

**1. Create the workspace:**
```bash
mkdir -p ~/ws_fastlio/src
cd ~/ws_fastlio/src
```

**2. Clone the Fast-LIO ROS 2 Port:**
```bash
git clone https://github.com/Ericsii/FAST_LIO_ROS2.git --recursive
```

**3. Configure your Sensor Topics:**

Before building, update the config to match our Gazebo bag topics.

Open `~/ws_fastlio/src/FAST_LIO_ROS2/config/avia.yaml` (or whichever config your launch file targets) and update these lines:

```yaml
lidar_topic:  "/lidar/points"
imu_topic:    "/imu/data"
```

**4. Build Fast-LIO2:**
```bash
cd ~/ws_fastlio
colcon build --symlink-install
```

---

## Phase 3: Workspace Overlaying (`.bashrc`)

To ensure your terminal always finds the packages in the correct order, append these lines to the bottom of your `~/.bashrc` file. **The order is strictly mandatory.**

```bash
# Open .bashrc
vim ~/.bashrc

# Add these to the very bottom:
source /opt/ros/humble/setup.bash       # Replace 'humble' with your ROS 2 distro
source ~/ws_livox/install/setup.bash
source ~/ws_fastlio/install/setup.bash
source ~/auto-drone-mapping/install/setup.bash
```

Save the file, then restart your terminal or run `source ~/.bashrc`.

---

## Phase 4: The Extended Kalman Filter (EKF)

The EKF acts as a mathematical referee, smoothing the highly accurate local odometry from Fast-LIO2 with the baseline odometry from the PX4 flight controller.

**1. Install the `robot_localization` package:**
```bash
sudo apt install ros-$ROS_DISTRO-robot-localization
```

**2. Create the Configuration File:**

In your main workspace (`~/auto-drone-mapping/src/your_package/config/`), create `ekf.yaml`:

```yaml
ekf_filter_node:
  ros__parameters:
    frequency: 30.0
    sensor_timeout: 0.1
    two_d_mode: false
    publish_tf: true
    map_frame: map
    odom_frame: odom
    base_link_frame: body  # Matches RTAB-Map frame_id
    world_frame: odom

    # Source 1: Fast-LIO2 Odometry
    odom0: /Odometry
    odom0_config: [true,  true,  true,   # X, Y, Z
                   true,  true,  true,   # Roll, Pitch, Yaw
                   false, false, false,  # X_vel, Y_vel, Z_vel
                   false, false, false,  # Roll_vel, Pitch_vel, Yaw_vel
                   false, false, false]  # Accel

    # Source 2: PX4 SITL Odometry (Update topic name as needed)
    odom1: /fmu/out/vehicle_odometry
    odom1_config: [false, false, false,
                   false, false, false,
                   true,  true,  true,
                   true,  true,  true,
                   false, false, false]
```

## Phase 5: Rewiring RTAB-Map

We need to tell RTAB-Map to stop guessing its location using visual odometry and instead trust the fused odom coming out of our EKF.

Update your RTAB-Map launch command:

```bash
ros2 launch rtabmap_launch rtabmap.launch.py \
    rtabmap_args:="--delete_db_on_start" \
    visual_odometry:=false \
    odom_topic:=/Fused_odom \
    frame_id:="body" \
    rgb_topic:=/camera/color/image_raw \
    depth_topic:=/camera/depth/image_raw \
    camera_info_topic:=/camera/color/camera_info \
    subscribe_scan_cloud:=true \
    scan_cloud_topic:=/lidar/points \
    approx_sync:=true \
    qos:=2 \
    use_sim_time:=true
```

## Phase 6: Launching the Full Pipeline

To test the entire SLAM stack, open **separate terminals** (ensure your `.bashrc` has sourced all workspaces) and run the following commands:

### Terminal 1 — Virtual Drone (Bag Playback)

> ⚠️ **Important:** Remove the IMU remap flag from your previous command so Fast-LIO2 can receive raw acceleration data!

```bash
ros2 bag play ~/dev/auto-drone-mapping/bag_files/drone_warehouse_bag_0.db3 -s sqlite3 --clock
```

### Terminal 2 — Fast-LIO2 (LiDAR + IMU Odometry)

```bash
ros2 launch fast_lio mapping.launch.py config_file:=avia.yaml use_sim_time:=true
```

### Terminal 3 — Extended Kalman Filter (Sensor Fusion)

```bash
ros2 run robot_localization ekf_node --ros-args \
    --params-file ~/auto-drone-mapping/src/your_package/config/ekf.yaml \
    --remap odometry/filtered:=/Fused_odom
```

### Terminal 4 — RTAB-Map (The SLAM Brain)

```bash
# Run the modified launch command from Phase 5 here
ros2 launch rtabmap_launch rtabmap.launch.py \
    rtabmap_args:="--delete_db_on_start" \
    visual_odometry:=false \
    odom_topic:=/Fused_odom \
    frame_id:="body" \
    rgb_topic:=/camera/color/image_raw \
    depth_topic:=/camera/depth/image_raw \
    camera_info_topic:=/camera/color/camera_info \
    subscribe_scan_cloud:=true \
    scan_cloud_topic:=/lidar/points \
    approx_sync:=true \
    qos:=2 \
    use_sim_time:=true
```

### Terminal 5 — RViz2 (Live Visualizer)

```bash
ros2 run rviz2 rviz2 --ros-args -p use_sim_time:=true
```
